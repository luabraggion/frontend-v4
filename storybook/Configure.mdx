# 🚀 Sistema de Componentes - Next.js

Bem-vindo à documentação interativa do sistema de componentes deste projeto! Aqui você encontra tudo sobre arquitetura, padrões, exemplos, guidelines, comandos e dicas para evoluir e manter o frontend com qualidade e performance.

---

## 📦 Stack Tecnológica

- **Next.js 15** (App Router, SSR/SSG, Turbopack)
- **React 19** (Concurrent, Server Components)
- **TypeScript 5** (Type Safety)
- **Tailwind CSS 4** (Utility-first CSS)
- **shadcn/ui** (Componentes acessíveis)
- **Lucide React** (Ícones SVG)
- **Zustand** (State Management)
- **Axios** (HTTP Client)
- **next-themes** (Temas dinâmicos)
- **ESLint, Prettier, Husky** (Qualidade de código)

## 🎨 Componentes

Os componentes são organizados por categoria para facilitar a escalabilidade e manutenção. Todos são escritos em TypeScript, memoizados e seguem padrões de acessibilidade e responsividade.

**Principais categorias:**

- `ui/`: Base shadcn/ui (Button, Input, Dialog, Calendar, Tooltip...)
- `forms/`: Campos e labels de formulários
- `buttons/`: Botões customizados
- `navigation/`: Breadcrumbs, Paginação
- `feedback/`: Alertas, Tooltips, Avatares
- `layout/`: Header, Footer

**Exemplo de uso:**

```tsx
import { TooltipWrapper, Avatar } from '@/components/feedback';
import { Input, DatePicker } from '@/components/forms/Fields';
import { ButtonWithIcon } from '@/components/buttons';

<TooltipWrapper content="Dica útil">
  <ButtonWithIcon icon={<Save />}>Salvar</ButtonWithIcon>
</TooltipWrapper>;
```

Consulte o README de cada subpasta para detalhes e exemplos específicos.

## 🔄 State Management & Hooks

- **Zustand** para stores centralizados (`store/`)
- **Custom hooks** para lógica reutilizável (`hooks/`)

**Exemplo de hook:**

```tsx
const useUser = () => {
  const user = useUserStore((s) => s.user);
  // ... lógica customizada
  return user;
};
```

## 📝 Convenções & Boas Práticas

- Componentes em PascalCase, hooks em camelCase
- Sempre usar TypeScript e tipar props
- Memoização (`memo`, `useMemo`, `useCallback`)
- Barrel exports (`index.ts`)
- Imports organizados (externos, internos, relativos)
- Classes Tailwind sempre por último no `className`
- Documentação e exemplos em todos os componentes
